# Kramer JS

TypeScript/React port of the Kramer autolayout engine.

## Packages

| Package | Description | Tests |
|---------|-------------|-------|
| [@kramer/core](packages/core/) | Schema, layout pipeline, expression language, query state | 113 |
| [@kramer/measurement](packages/measurement/) | Browser and approximate text measurement strategies | 7 |
| [@kramer/graphql](packages/graphql/) | GraphQL query parsing, AST expansion, introspection | 13 |
| [@kramer/react](packages/react/) | React renderer (AutoLayout, Table, Outline, ColumnHeader) | 17 |
| [@kramer/react-ui](packages/react-ui/) | Interaction panels (FieldSelector, FieldInspector) | 7 |

## Quick Start

```bash
# Install dependencies
pnpm install

# Run all tests
pnpm run -r test

# Run the demo
pnpm --filter @kramer/react dev

# Build all packages
pnpm run -r build
```

## Architecture

The core pipeline is pure TypeScript with zero DOM dependency:

1. **Schema** — `Relation`/`Primitive` discriminated union tree
2. **Measure** — P90 statistical content-width measurement via `MeasurementStrategy`
3. **Layout** — `ExhaustiveConstraintSolver` chooses table vs outline per relation
4. **Justify** — two-pass minimum-guarantee width distribution
5. **Compress** — DP-optimal column partitioning (painter's partition)

The React renderer consumes the pipeline output:

- `<AutoLayout>` — ResizeObserver + solver + mode switching
- `<Table>` / `<VirtualOutline>` — table and outline rendering
- `<ColumnHeader>` — sort indicators, vertical rotation
- `@tanstack/react-virtual` — virtualized scrolling

## Contract Tests

Shared JSON fixtures at `test/fixtures/` (repo root) ensure Java and TypeScript pipelines produce identical layout decisions. Generated by Java `ContractFixtureGeneratorTest`, validated by TS `contract.test.ts`.

## Maven Integration

Tests run via `mvn test -pl :kramer-js` using `frontend-maven-plugin` with corepack + pnpm.
