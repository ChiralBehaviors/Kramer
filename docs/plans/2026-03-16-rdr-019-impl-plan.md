## Executive Summary

Implement Tufte sparkline rendering for array-valued numeric `Primitive` fields. Sparklines are word-sized inline charts (Polyline + optional IQR band + end marker) rendered in VirtualFlow cells via `PrimitiveSparklineStyle`. No schema-layer change — detection is purely in `PrimitiveLayout.measure()`. `SparklineStats` is a nullable 4th sub-record on `MeasureResult`.

Files: `kramer/src/main/java/com/chiralbehaviors/layout/style/PrimitiveRenderMode.java`

Steps:

2. Add `SPARKLINE` to the enum.
3. Verify all existing switch statements compile (no switch exhaustion failures — existing code uses `default` or non-exhaustive switches).

**Bead Kramer-woy — 019-P1-T2: Add SparklineStats record to MeasureResult**

Files: `kramer/src/main/java/com/chiralbehaviors/layout/style/MeasureResult.java`

Steps:
1. Write test: `MeasureResult` with `null` `sparklineStats` is valid; `SparklineStats` fields are accessible.
2. Define `SparklineStats` record (can be a nested record or top-level in package):
   ```java
   record SparklineStats(
       double seriesMin,
       double seriesMax,
       int maxSeriesLength,
       double q1,
       double q3
   ) {}
   ```
3. Add `SparklineStats sparklineStats` as the 4th nullable sub-record parameter to `MeasureResult` (after `pivotStats`).
4. Update compact constructor and any factory methods; pass `null` for existing call sites.

Research: 019-research-7 (position confirmed: 4th sub-record after contentStats, numericStats, pivotStats).

**Bead Kramer-l6d — 019-P1-T3: Bifurcate array-vs-scalar detection in PrimitiveLayout.measure()**

Files: `kramer/src/main/java/com/chiralbehaviors/layout/PrimitiveLayout.java`

CRITICAL (research 019-research-6): The current numeric detection loop conflates scalar-numeric (→BAR) and array-of-numbers (→SPARKLINE). The fix must bifurcate correctly.

Steps:
1. Write tests:
   - Array of numbers data → `renderMode == SPARKLINE`
   - Scalar number data → `renderMode == BAR` (unchanged)
   - Array of strings data → not SPARKLINE (falls through to list rendering)
   - Mixed numeric/non-numeric array → SPARKLINE if >50% numeric, else text
2. Implement `allElementsNumeric(ArrayNode node)` helper: returns true iff all elements `isNumber()`.
3. In the `isArray()` branch of `measure()`: when `allElementsNumeric(datum)` is true, set `renderMode = SPARKLINE`; do NOT set BAR.
4. Accumulate `SparklineStats` across sample rows when SPARKLINE:
   - Flatten all element values across all rows into a list.
   - `seriesMin` / `seriesMax` = global min/max.
   - `maxSeriesLength` = longest array.
   - `q1` / `q3` = 25th/75th percentile across all flattened values (not per-series medians).
5. Add `case SPARKLINE` to the `renderModeOverride` switch for explicit stylesheet opt-in.
6. Construct `SparklineStats` and store in `MeasureResult`.

Research: 019-research-2 (isArray branch confirmed), 019-research-6 (bifurcation critical).

### Phase 2: Sparkline Cell Rendering

**Goal**: Visual sparkline in VirtualFlow cells. Activates when `renderMode == SPARKLINE`.

**Bead Kramer-t92 — 019-P2-T1: Implement PrimitiveSparklineStyle and SparklineCell**

Files (new):
- `kramer/src/main/java/com/chiralbehaviors/layout/style/PrimitiveSparklineStyle.java` (static inner class of `Style`, or standalone — match `PrimitiveBarStyle` pattern)
- `kramer/src/main/resources/com/chiralbehaviors/layout/style/sparkline.css`

Steps:
1. Write tests: coordinate mapping (series values → pixel x/y), downsampling (series > width → reduces to width points), null/NaN gap handling, short series (< 4 values) suppresses band, fallback to text for non-array.
2. Implement `PrimitiveSparklineStyle extends PrimitiveStyle`:
   - `width()` returns `0.0` (fills justifiedWidth like `PrimitiveBarStyle`).
   - `build(FocusTraversal<?> pt, PrimitiveLayout layout)` returns a new `SparklineCell`.
3. `SparklineCell extends PrimitiveLayoutCell<StackPane>` (StackPane root — research 019-research-3; Canvas extends Node not Region, would fail type bound):
   - Fields: reference to `PrimitiveLayout` for `getMeasureResult().sparklineStats()`.
   - `updateItem(JsonNode item)`:
     - If `item` is null, not an array, or series is empty → fall back to text (set label text).
     - `double[] values = extractNumericSeries(item)` — skip non-numeric elements.
     - `SparklineStats stats = layout.getMeasureResult().sparklineStats()`.
     - Compute x: evenly spaced `0 .. justifiedWidth` for `values.length` points (or downsampled count).
     - Compute y: map `stats.seriesMin()..stats.seriesMax()` → `cellHeight..0` (y-axis inverted).
     - Downsampling: if `values.length > (int) justifiedWidth`, LTTB or simple decimation to `(int) justifiedWidth` points.
     - `Polyline line` from x/y pairs (skip NaN gaps by splitting into segments if needed).
     - `Rectangle band` at y(q3)..y(q1), full width; hide if series length < 4.
     - `Circle endDot` at (`xLast`, `yLast`), radius 1.5; the `lastValue` is `values[values.length - 1]`.
     - `getChildren().setAll(band, line, endDot)`.
4. Create `sparkline.css`: define `.sparkline-line` stroke color, `.sparkline-band` fill color, `.sparkline-marker` fill color.

Research: 019-research-1 (build signature: FocusTraversal, PrimitiveLayout — data via updateItem()), 019-research-3 (StackPane root), 019-research-5 (inner class pattern).

**Bead Kramer-71q — 019-P2-T2: Wire SPARKLINE dispatch in Style.buildControl() and computePrimitiveStyle()**

Files: `kramer/src/main/java/com/chiralbehaviors/layout/style/Style.java`, `kramer/src/main/java/com/chiralbehaviors/layout/PrimitiveLayout.java`

Steps:
1. Write tests: `renderMode == SPARKLINE` → `computePrimitiveStyle()` returns `PrimitiveSparklineStyle`; `clear()` nulls `cachedSparklineStyle`.
2. In `Style.computePrimitiveStyle()` (currently returns `PrimitiveTextStyle` unconditionally — research 019-research-4): add `case SPARKLINE -> new PrimitiveSparklineStyle(...)` (or cached allocation).

Research: 019-research-4 (computePrimitiveStyle extension point), 019-research-5 (cached style pattern + clear() requirement).

## Key Design Constraints (from gate review)

- `lastValue` is NOT in `SparklineStats` — per-cell computation in `updateItem()` via `values[values.length - 1]`.
- No `PrimitiveValueType` schema enum — detection in `measure()` only.
- `q1`/`q3` flattened across all rows (not per-series medians) — global IQR band.
- `SPARKLINE` must preempt `averageCardinality > 1` guard in `buildControl()`.
- `cachedSparklineStyle` must be null-reset in `clear()`.
- `SparklineStats` is the 4th nullable sub-record (after `contentStats`, `numericStats`, `pivotStats`).
- Cell root must be `StackPane` (extends Region), not `Canvas` (extends Node — fails LayoutCell type bound).
- `build()` signature is `(FocusTraversal<?>, PrimitiveLayout)` — data arrives via `updateItem()`, not `build()`.

## Risks

| Risk | Severity | Mitigation |
|---|---|---|
| Polyline perf at >500 points | Low | Downsample to one point per horizontal pixel |
| Non-numeric mixed in array | Low | Skip non-numeric; if >50% non-numeric, fall back to text |
| Null/NaN values in series | Medium | NaN convention: gap in Polyline (split into segments) |
| Ragged series lengths | Low | `maxSeriesLength` normalizes x-axis; short series x-compressed |
| Quartile on <4 values | Low | Suppress band when series.length < 4 |

---

## Success Criteria

1. Array-valued numeric fields render as word-sized inline sparklines.
2. Sparkline height matches surrounding text height.
3. IQR band visible by default.
4. End marker visible by default.
5. Min/max values globally normalized across all rows in column.
6. LayoutStylesheet controls all visual parameters per SchemaPath.
7. Non-numeric arrays fall back gracefully.
8. All existing tests pass (sparkline is opt-in).
