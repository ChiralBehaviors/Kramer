# RDR-013 Implementation Plan: Statistical Content-Width Measurement

**Epic**: Kramer-w16 (P1)
**RDR**: docs/rdr/RDR-013-statistical-content-width-measurement.md
**nx memory**: Kramer_rdr/013 (status: accepted)
**Date**: 2026-03-15

## Executive Summary

Implement SIEUFERD-style statistical content-width measurement in PrimitiveLayout.
Phase 1 replaces `averageWidth` with `p90Width` for variable-length columns, producing
tighter layouts without CSS tuning. Phase 2 adds convergence detection and freezing for
layout stability. Phase 1 has no external blockers. Phase 2 depends on Kramer-517
(LayoutStylesheet wired into measure()).

## Risk Register

| Risk | Severity | Mitigation |
|------|----------|------------|
| p90 on misclassified fixed-length column causes truncation | High | p90 only in isVariableLength==true branch; inherited heuristic, not introduced by this RDR |
| Phase 2 blocked on Kramer-517 (LayoutStylesheet wiring) | Medium | Phase 1 delivers value independently; Kramer-517 is a shared prereq for RDR-014/015/018 |
| Convergence params (epsilon, k) need tuning per field type | Low | Exposed as LayoutStylesheet properties; defaults are conservative |
| Width oscillation before convergence | Medium | Monotonic width (never shrink) can be added as follow-on if observed |

## Success Criteria (from RDR-013)

1. Column widths adapt to content without CSS tuning
2. Convergence within first 30 unique values
3. No layout width oscillation after convergence
4. P90-based widths produce visually tighter layouts than average-based widths
5. Zero performance regression on measure()
6. All 142+ existing tests pass
7. Fixed-length columns display without truncation
