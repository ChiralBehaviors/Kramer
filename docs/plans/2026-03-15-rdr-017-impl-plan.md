## Phase 1 — LayoutSearch data model (Kramer-41y)

**Effort**: LOW. Zero prerequisites.

### New files

| File | Purpose |
|------|---------|
| `kramer/src/main/java/com/chiralbehaviors/layout/LayoutSearch.java` | Main search engine |
| `kramer/src/main/java/com/chiralbehaviors/layout/SearchResult.java` | `record SearchResult(SchemaPath path, int rowIndex, String matchedValue, int matchStart, int matchLength)` |
| `kramer/src/main/java/com/chiralbehaviors/layout/NavigationPath.java` | `record NavigationPath(List<NavigationStep> steps)` + nested `record NavigationStep(SchemaPath field, int rowIndex)` |
| `kramer/src/test/java/com/chiralbehaviors/layout/LayoutSearchTest.java` | **Write first (TDD)** |

### LayoutSearch public API

```java
public class LayoutSearch {
    public LayoutSearch(AutoLayout layout) { ... }
    public void setQuery(String query);
    public String getQuery();
    public boolean isCaseSensitive();
    public void setCaseSensitive(boolean v);
    public boolean isWrapAround();
    public void setWrapAround(boolean v);
    public Optional<SearchResult> findNext();
    public Optional<SearchResult> findPrevious();
    public int countMatches();
}
```

### Algorithm

Schema depth-first traversal over `Relation.children()` -> `Primitive` fields. For each data row in `AutoLayout.getData()` (which is an array `JsonNode`), for each `Primitive` in schema order, call `SchemaNode.asText(node)` and test `contains(query)` (or `equalsIgnoreCase` normalised). Empty/blank query returns `Optional.empty()` immediately.

### Test commands

```
mvn -pl kramer test -Dtest=LayoutSearchTest
```

### Test cases (write before implementation)

- `findNext()` returns `Optional.empty()` when query is null or blank
- `findNext()` returns first matching `SearchResult` with correct `rowIndex` and `matchStart`
- `findPrevious()` wraps around from row 0 when `wrapAround=true`
- `findPrevious()` stops at row 0 when `wrapAround=false`
- `countMatches()` returns total across all rows and fields
- Case-insensitive mode matches regardless of case

## Phase 2b — Nested VirtualFlow chain navigation (Kramer-2ff)

**Effort**: MEDIUM-HIGH. Depends on: Kramer-npg.

### Modified files

| File | Change |
|------|--------|
| `kramer/src/main/java/com/chiralbehaviors/layout/LayoutSearch.java` | Add `navigateToMatch(NavigationPath)`, `navigateStep(NavigationPath, int)`, `volatile boolean navigationCancelled`, `boolean navigationInProgress` |
| `kramer/src/main/java/com/chiralbehaviors/layout/AutoLayout.java` | Add `resolveVirtualFlow(int depth)` (fresh scene-graph walk); `autoLayout()` sets `navigationCancelled=true` on attached `LayoutSearch` |

### Async chain pattern

```java
void navigateStep(NavigationPath path, int depth) {
    if (depth >= path.steps().size() || navigationCancelled) return;
    NavigationStep step = path.steps().get(depth);
    VirtualFlow<?> vf = autoLayout.resolveVirtualFlow(depth);  // fresh every hop
    if (vf == null) return;  // stale reference guard
    vf.show(step.rowIndex());
    Platform.runLater(() -> navigateStep(path, depth + 1));
}
```

### Cancellation invariants

- `volatile boolean navigationCancelled` — set by `autoLayout()` rebuild; checked at top of each `navigateStep` hop
- `boolean navigationInProgress` — prevents overlapping chains; new `F3` press cancels current chain before starting
- VirtualFlow references **must not** be captured across `runLater` boundaries — resolve fresh each hop

### Known limitation

`CursorState.fieldPath` is always `null` ("not yet implemented" comment in source). Phase 2b scrolls to the correct row at each nesting depth but cannot place the cursor at the exact cell within a nested row. Exact-cell focus in nested tables is deferred until RDR-005 implements `fieldPath`.

## Out of Scope (RDR-017b)

**Phase 4 — Match highlighting via TextFlow** is deferred. It requires converting `Primitive` cells from `Label` to `TextFlow` with pre-match/match/post-match `Text` runs. This touches the shared cell rendering pipeline and is MEDIUM effort. Track as a separate task or RDR-017b after Phases 1–3 are stable.

## Risk Register

| Risk | Severity | Mitigation |
|------|----------|------------|
| Nested VF async chain invalidated by autoLayout() rebuild | HIGH | Crash guard: resolve VF fresh each hop; cancellation flag |
| ENTER key conflict with TRAVERSAL_INPUT_MAP | MEDIUM | F3 for find-next; TextField.onAction handles Enter within bar |
| CursorState.fieldPath always null | MEDIUM | Phase 2b scoped to row-level scroll only; RDR-005 delivers exact cell |
| Search on large datasets | LOW | O(rows × fields) string matching; 10K × 20 fields is fast |
