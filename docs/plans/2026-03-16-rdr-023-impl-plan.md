# RDR-023: Reactive Layout Invalidation — Implementation Plan

**Date**: 2026-03-16
**RDR**: docs/rdr/RDR-023-reactive-layout-invalidation.md
**Design spec**: docs/plans/2026-03-16-rdr-012-phase3-reactive-invalidation-spec.md
**Epic**: Kramer-p22

---

## Summary

Close two cache-invalidation gaps introduced by RDR-016:
1. `frozenResult` in `PrimitiveLayout` is never cleared on data change (only on stylesheet change).
2. `decisionCache` in `AutoLayout` has no data-content key component; stale LayoutResults survive data updates.

The fix is a three-step pipeline inserted into the `setContent()` hot path: detect changed paths (Step A), selectively re-measure changed leaves (Step B), detect mode flip via threshold comparison (Step C). Phases 3a+3b ship as a correctness fix; Phase 3c is an enhancement.

---

## Phases and Beads

### Prerequisite: FXAT Guard

**Bead**: Kramer-o9v
**Status**: open
**Blocks**: Kramer-eyr (Phase 3a)

Standalone correctness fix. Add `Platform.isFxApplicationThread()` assertion at the top of `setContent()`. If off-FXAT, enqueue via `Platform.runLater()` and return. All snapshot state added in Phase 3a is only safe under FXAT serialization; this guard must land first.

Affected file: `AutoLayout.java:617` (`setContent()`)

---

### Phase 3a: Data Snapshot + frozenResult Invalidation

**Bead**: Kramer-eyr
**Depends on**: Kramer-o9v
**Status**: open

New state added to `AutoLayout`:
- `Map<SchemaPath, List<String>> dataSnapshot` — string-rendered values per primitive path from the last successful `setContent()`.
- `Map<SchemaPath, Double> p90Snapshot` — p90 width read from `frozenResult.p90Width()` before clearing; populated inside `detectChangedPaths()`.

New method in `PrimitiveLayout`:
- `void clearFrozenResult()` — package-accessible; sets `frozenResult = null`.

New method in `AutoLayout`:
- `Set<SchemaPath> detectChangedPaths(JsonNode newData)` — walks schema tree, extracts `asText()` values per `Primitive` path, compares against `dataSnapshot`, records old p90 into `p90Snapshot` before clearing, returns changed paths.

Snapshot update: both maps updated as a transaction at end of successful pipeline. Try/finally: finally block clears both maps on exception (self-healing; first call after clearing handles empty snapshot correctly).

Schema root change: clear `dataSnapshot`, `p90Snapshot`, and `decisionCache` when new `SchemaNode` assigned.

Row deletion: if incoming data has fewer rows than snapshot for a path, treat as changed; trim snapshot entries.

TDD (write tests before implementation):
1. Single-field update — only that path's `frozenResult` cleared
2. Multi-field update — both cleared, unrelated paths untouched
3. Same-value no-op — no `frozenResult` cleared
4. New row appended — all paths cleared (size mismatch)
5. Row deleted — affected paths cleared, snapshot trimmed
6. Schema change — all three maps cleared

### Phase 3b: Selective Re-measure with Bucket Comparison

**Bead**: Kramer-bhin
**Depends on**: Kramer-eyr
**Status**: open

New method in `AutoLayout`:
- `Map<SchemaPath, Double> remeasureChanged(Set<SchemaPath> changedPaths, JsonNode data)` — invokes `measure()` on each changed `PrimitiveLayout`; returns only paths where bucket changed.

Bucket comparison (EPSILON): `(int)(new_p90 / 10) != (int)(old_p90 / 10)`. Old p90 read from `p90Snapshot`. A shift within the same bucket cannot change any `LayoutDecisionKey`-keyed cache result.

Outline sibling-lateral expansion: if changed `PrimitiveLayout`'s parent `RelationLayout` is in OUTLINE mode, expand re-measure set to all siblings at that level before invoking `measure()`.

Hot path integration in `setContent()`:
1. `detectChangedPaths()` → `changedPaths`
2. If `changedPaths` empty → `control.updateItem()` only (case (a) fast path, `<2ms`)
3. `remeasureChanged(changedPaths, data)` → `bucketChangedPaths`
4. If `bucketChangedPaths` empty → `control.updateItem()` only
5. Else → proceed to Phase 3c

TDD (write tests before implementation):
1. Value within same bucket — `remeasureChanged` returns empty map
2. Value crosses bucket boundary — returns entry with new p90
3. Outline sibling expansion — all siblings re-measured
4. Shrink within same bucket — empty map
5. Fast-path short-circuit — `detectChangedPaths` empty; `measure()` never called

### Phase 3c: Lightweight Mode-Flip Detection (Enhancement)

**Bead**: Kramer-nyqy
**Depends on**: Kramer-bhin
**Status**: open

This is NOT full partial constraint re-solve (requires RDR-012 Phase 1). This is a lightweight threshold check: compare new p90 sum against cached `LayoutResult.tableColumnWidth()` to detect whether a mode flip occurred.

New method in `AutoLayout`:
- `boolean detectModeFlip(Map<SchemaPath, Double> changedP90s)` — for each changed path, look up the cached `LayoutResult` for the corresponding `RelationLayout` from `decisionCache`. Compare updated p90 widths against `tableColumnWidth()`. Return `true` if any mode assignment would change.

Hot path continuation (after Phase 3b step 5):
- `detectModeFlip` returns `false` → `control.updateItem()` only (no rebuild)
- `detectModeFlip` returns `true` → rebuild affected subtree (LCA node `buildControl()`), then `control.updateItem()`

`decisionCache` interaction: after rebuild, update cache entries for affected subtree nodes. Root and sibling subtrees retain existing entries.

TDD (write tests before implementation):
1. No bucket changes — `detectModeFlip` not called; only `updateItem()` runs
2. Bucket changes, mode unchanged — returns false; only `updateItem()` runs
3. Bucket changes, mode flips — returns true; affected subtree `buildControl()` called
4. Multiple leaves in different subtrees — LCA = root; full rebuild triggered

## Key Invariants

- EPSILON = `(int)(new_p90 / 10) != (int)(old_p90 / 10)` (bucket identity, not magnitude)
- `p90Snapshot` populated BEFORE `clearFrozenResult()` is called
- Snapshot updates are transactional: try/finally clears both on exception
- `decisionCache` cleared on schema root change
- `dataSnapshot`, `p90Snapshot`, `decisionCache` all cleared together on schema change
- All snapshot mutations are FXAT-only (enforced by Kramer-o9v)
